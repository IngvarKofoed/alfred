Added POSTGRES_URL (optional) to the shared config. 8. Implemented build-order step 3 (agent core, as a library): packages/agent-core with the hand-rolled loop, the LlmProvider streaming abstraction, a GeminiProvider on @google/genai (streaming + function-calling), and the Tool interface + a built-in echo tool. Added GEMINI_API_KEY (optional) + DEFAULT_MODEL (gemini-2.5-flash) to the shared config. A fake-provider unit test covers the full tool-call round-trip offline; a live Gemini integration test + CLI runner are gated on GEMINI_API_KEY. Build/lint/offline tests green; the worker/pg-boss/NOTIFY/SSE/web pipeline is deferred to the next step.
9. Wrote docs/specs/2026-06-02-chat-pipeline.md — spec for the end-to-end text-only chat pipeline (worker + pg-boss + NOTIFY/SSE + web).
10. Implemented the chat pipeline (text-only, no tools): new alfred-worker consumes pg-boss jobs, runs the agent-core Gemini loop, persists the assistant turn, and streams tokens via Postgres NOTIFY; webserver gained POST message / history / SSE-stream routes that LISTEN and forward; added the agent_runs table (+ one-active-run-per-conversation index) and pg-boss (retryLimit 0 + long expiry) in packages/db; messages.content persists the agent-core ContentPart[]; a fixed owner user is upserted on first message (resolves the step-2 seeding deferral); the web client is now a streaming chat (hand-rolled EventSource); worker boot sweeps orphaned 'running' runs to 'failed'. Worker integration test (fake provider) verifies the worker→loop→persistence→status path against Postgres; real end-to-end chat needs GEMINI_API_KEY. ecosystem.config.cjs gained alfred-worker.
11. Fixed loadConfig (packages/shared) to load the repo-root .env regardless of cwd — it walks up to the pnpm-workspace.yaml root and loads its .env, so `pnpm --filter <pkg> dev` picks up a single root .env. Exported env vars still take precedence.
12. Documented GEMINI_API_KEY + DEFAULT_MODEL in .env.example (the LLM keys added in step 3).
13. Renamed the config key DEFAULT_MODEL -> GEMINI_MODEL (provider-scoped, so other providers can add OPENAI_MODEL/etc.). Updated shared config, the GeminiProvider, and .env.example.
